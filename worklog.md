## 2026-07-14 18:13 — вместе с Claude Code

Пользователь протестировал первую версию responsive на реальных
устройствах и указал на 2 серьёзные проблемы: (1) на мобильном вся
страница, включая шапку, скроллилась вместе — нужно было развести
лого/тему/логаут на первую строку, меню на вторую, и обе строки
закрепить неподвижно; (2) на планшетах высота левой колонки "гуляла"
между устройствами — то обрезалась, то с лишней прокруткой.

Причина обеих проблем — использование `100vh` и `position: sticky`.
`100vh` не учитывает динамическую панель браузера на мобильных/
планшетах (адресная строка то показывается, то прячется), поэтому
реальная видимая высота экрана отличается от расчётной — отсюда
"гуляющая" высота на разных устройствах. `position: sticky` не даёт
железной гарантии неподвижности при таких раскладках.

Переделали архитектуру целиком:
- `.app-shell` теперь `display: flex` (было `grid`) с `height: 100vh;
  height: 100dvh;` — `dvh` (dynamic viewport height) отслеживает
  реальную видимую высоту экрана, обновляясь вместе с
  показом/скрытием панели браузера; `vh` оставлена первой строкой как
  фолбэк для старых браузеров.
- `.sidebar` больше не `position: sticky` — просто растягивается на
  всю высоту `.app-shell` по умолчанию (flex `align-items: stretch`),
  что даёт гарантированно правильную высоту на любом устройстве без
  скролла самой колонки.
- `.main` стала `flex: 1 1 auto; min-height: 0; overflow-y: auto;` —
  теперь скроллится **только** контент, а не вся страница. `min-
  height: 0` обязателен здесь — без него flex-потомок с overflow:auto
  не сжимается и не скроллится сам, вместо этого растягивает и
  скроллит всю страницу (ровно тот баг, который был).
- Мобильный сайдбар (`≤640px`) теперь `flex-direction: column` на
  `.app-shell` (шапка сверху, контент под ней, а не сбоку), а внутри
  самого `.sidebar` — 2 ряда через `flex-wrap`: ряд 1 = лого + (через
  `margin-left: auto`) тема + логаут, ряд 2 = меню. Выяснилось, что
  простого `margin-left: auto` для переноса недостаточно — flex-wrap
  решает перенос по "естественному" размеру элементов, не учитывая,
  что auto-margin потом съест место, поэтому ссылки меню норовили
  остаться в первом ряду. Починили строгим переносом через
  `.sidebar::after { flex-basis: 100%; height: 0; }` — псевдоэлемент-
  разделитель, который гарантированно проталкивает всё, что идёт
  после него по `order`, на новую строку.
- Планшет (`≤1024px`) упростился — почти вся логика теперь
  общая (dvh-фикс), нужно было только сузить `.sidebar` до 180px.

Написал отдельный тест (`layout-check.js`) на 4 реальных viewport
(десктоп, iPad portrait 768×1024, iPad landscape 1024×700, iPhone
375×700), который проверяет программно, а не только "на глаз": нет
горизонтальной прокрутки; высота `.app-shell` точно равна высоте
окна; страница (`window.scrollY`) не скроллится при скролле колесом
над контентом; `.main` скроллится сама; сайдбар/шапка не сдвигается
после скролла контента; на мобильном ссылки меню реально переносятся
на вторую строку, а logout остаётся на первой. Все 22 проверки
прошли. Плюс перепрогнал полный функциональный 29-шаговый тест — тоже
всё зелёное, регрессий нет.

Закоммичено и запушено.
